# Copilot Instructions for Dual RAG Implementation

This file provides instructions for GitHub Copilot when working on this project. Copilot will follow these guidelines when generating code and suggestions.

---

## Code Generation Style

### Type Hints
- **Always include type hints** on all function parameters and return types
- Use `typing` module for complex types: `List`, `Dict`, `Tuple`, `Optional`
- Example:
  ```python
  def retrieve(self, query_embedding: np.ndarray, top_k: int = 3) -> List[Dict]:
  ```

### Error Handling
- **Always wrap HTTP calls in try-except blocks**
- Catch specific exceptions: `ConnectionError`, `Timeout`, `JSONDecodeError`
- Provide user-friendly error messages with context
- Example:
  ```python
  try:
      response = requests.post(url, json=payload, timeout=10)
  except requests.exceptions.ConnectionError as e:
      print(f"Failed to connect to ResonanceDB: {e}")
      return None
  ```

### Code Organization
- Use **class-based design** where possible (e.g., `BaselineRetriever`, `ResonanceDBClient`)
- Keep methods focused and single-purpose
- Use **docstrings** for all classes and public methods (Google-style)
- Group related constants at module top level

### Naming Conventions
- `snake_case` for variables and functions
- `PascalCase` for classes
- `UPPER_CASE` for constants
- Private methods: `_method_name`

### Documentation
- Include docstrings with Args, Returns, Raises sections
- Add inline comments for complex logic
- Use type hints inside docstrings if clarification needed

---

## Project-Specific Conventions

### ResonanceDB Integration
- Base URL: `http://localhost:8080`
- Always check health before operations
- Handle dimension mismatches with clear error messages
- Return format: `List[Dict[str, any]]` with keys: `id`, `text`, `score`

### Embedding Models
- Model: `sentence-transformers/all-MiniLM-L6-v2`
- Dimension: 384
- Convert numpy arrays to lists before sending to HTTP endpoints

### Wave Mapping
- Amplitude: normalized absolute values (range [0, 1])
- Phase: random values in range [-π, π]
- Phase heuristic: shift by π if text contains negation words
- Negation words: "not", "never", "unless", "contraindicated", "no", "none", "without"

### ChromaDB Baseline
- Use ephemeral (in-memory) client for testing
- Metric space: `cosine`
- Store chunks as documents with metadata

## Special Notes

- **Docker Requirement:** ResonanceDB server must be running at `localhost:8080`
  - If not running, code should provide clear error message with setup instructions
  
- **PDF Loading:** Uses PyPDF2 (simplicity) or langchain (more features)
  - Fallback to hardcoded sample chunks if PDF fails
  
- **Batch Processing:** ResonanceDB supports batch insert but handle partial failures
  - Always log which records succeeded/failed
  
- **Phase Randomization:** Each vector-to-wave call generates **new random phase**
  - This is intentional for testing interference patterns
  - If deterministic phase needed for testing, add `seed` parameter
